SQL Injection

What is SQL Injection?

One of the most exploited vulnerabilities of SQL databases is an attack known as SQL Injection

This technique user inputs that hve not been properly verified before being sent back to the server

The best way to prevent an application from being vulnerable to SQL Injection is to Sanitize any user inputs, both on the front-end and the back-end

User's have permission to conduct SQL Injection attacks on this application, but bear in mind that using this technique in the wild is almost always illegal

SQL Injection attacks are performed by treating User Input as part of an SQL statement

Users add small bits of SQL code to their inputs while monitoring the results

Through this simple method, the attacker can learn what implementation of SQL is being used

The consequences of such an attack can be huge, and attackers can potentially use unsanitized inputs to gain access to sensitive data and potentially to the server the application is running on

An SQL Injection Attack can be performed by manipulating a SELECT query and making it return True no matter what condition was

A simple SQL Injection Attack usually takes the following form:

SELECT FROM dbTable WHERE key=0 OR 1=1

The example above has a condition, WHERE key=0 or 1=1. Since 1=1 is always true, the database will return each row in the table
